Маршрут находит метод — но что этот метод обязан вернуть? Разбираемся с контроллерами и
объектом `Response`.

## Контроллер — просто класс с методом

В прошлом уроке контроллеры уже были обычными PHP-классами без всякого наследования. Это не
случайность: единственное реальное требование Symfony — метод-экшн должен **вернуть объект
`Response`** (или его наследник). Всё остальное — дело вкуса и удобства.

На практике почти всегда наследуют `AbstractController` — он не добавляет требований, а даёт
готовые хелперы: `$this->render()` для Twig-шаблонов (следующий блок), `$this->json()` как
короткая запись для `JsonResponse`, `$this->redirectToRoute()`, `$this->createNotFoundException()`
и доступ к сервисам через автовайринг конструктора. Мы вернёмся к этим хелперам, когда они
понадобятся; пока пишем контроллеры как простые классы — так виднее, что происходит на самом
деле под капотом.

## Response — универсальный конверт для ответа

```php
use Symfony\Component\HttpFoundation\Response;

new Response(
    content: 'Тело ответа',
    status: 200,
    headers: ['X-Custom-Header' => 'value'],
);
```

Три составляющие любого HTTP-ответа — тело, код состояния и заголовки — все они здесь. Наследники
`Response` (например, `JsonResponse`, который встретим в следующем уроке) просто удобно
преднастраивают частный случай — тело в виде JSON с соответствующим `Content-Type`.

## Коды состояния — не только 200

Возвращать 200 на всё подряд — плохая практика. У `Response` есть именованные константы для
самых частых кодов:

```php
new Response('Создано', Response::HTTP_CREATED);      // 201
new Response('', Response::HTTP_NO_CONTENT);           // 204
new Response('Не найдено', Response::HTTP_NOT_FOUND);  // 404
new Response('Ошибка', Response::HTTP_BAD_REQUEST);    // 400
```

Код ответа — это часть контракта твоего API. На собеседовании про Symfony/REST почти наверняка
спросят разницу между 400 и 422, между 401 и 403 — это тема, к которой мы ещё вернёмся в блоке
про Security, но общий принцип уже сейчас: **код должен честно отражать, что произошло**,
клиент API ориентируется на него, а не только на текст тела.
