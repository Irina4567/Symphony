## Что такое роутинг

Роутинг — это сопоставление входящего запроса (метод + путь) конкретному методу конкретного
контроллера. В современном Symfony это делается PHP-атрибутом прямо над методом — никакого
отдельного файла конфигурации заводить не нужно:

```php
namespace App\Controller;

use Symfony\Component\HttpFoundation\Response;
use Symfony\Component\Routing\Attribute\Route;

class BookController
{
    #[Route('/books', name: 'books_list', methods: ['GET'])]
    public function list(): Response
    {
        return new Response('Список книг');
    }
}
```

Как только класс лежит в `src/Controller/` и зарегистрирован как сервис (а сервисы из `src/`
регистрируются автоматически — помнишь `App\: resource: '../src/'` из `config/services.yaml`?),
Symfony сам находит на нём методы с атрибутом `#[Route]` и строит из них таблицу маршрутов.
Проверить эту таблицу можно командой:

```bash
php bin/console debug:router
```

## Параметры пути

Часть пути можно сделать переменной — `{name}` в пути превращается в аргумент метода с тем же
именем:

```php
#[Route('/hello/{name}', name: 'hello', methods: ['GET'])]
public function hello(string $name): Response
{
    return new Response("Привет, {$name}!");
}
```

Symfony сам подставит значение из URL в переменную `$name` — конвертация типов (например, в
`int`) тоже происходит автоматически по тайп-хинту аргумента.

## Ограничения на параметры (requirements)

Чтобы `{id}` совпадал только с числами, а не с любой строкой, можно задать `requirements` прямо
в атрибуте:

```php
#[Route('/books/{id}', name: 'books_show', requirements: ['id' => '\d+'])]
public function show(int $id): Response { /* ... */ }
```

Без этого ограничения `/books/abc` тоже попытался бы попасть в этот метод — и упал бы с ошибкой
при попытке привести `"abc"` к `int`.

## methods: важно для REST

Один и тот же путь `/books` может обслуживать разные действия в зависимости от HTTP-метода —
`GET /books` (список) и `POST /books` (создание) вполне могут быть двумя разными методами
контроллера с одинаковым `path`, но разным `methods`. Это основа REST-подхода к API, которым мы
и будем пользоваться в BookShelf.

![Один путь /books — разные методы: GET ведёт в list(), POST ведёт в create()](/images/routing/one-path-different-methods.png)

## Генерация URL по имени

Жёстко прописывать пути строками по всему коду — плохая идея: изменишь путь в одном месте,
сломаешь ссылки в десяти других. Вместо этого маршруты генерируют по имени:

```php
$url = $this->generateUrl('books_show', ['id' => 42]); // "/books/42"
```

Это работает и в шаблонах Twig (`{{ path('books_show', {id: 42}) }}` — увидим в следующем блоке),
и в самих контроллерах, и в консольных командах.
