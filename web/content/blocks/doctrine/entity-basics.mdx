До сих пор книги BookShelf жили в обычном PHP-массиве прямо в контроллере — удобно для
демонстрации роутинга, но данные исчезали с каждым перезапуском, а поиск по ним был примитивным.
В этом блоке подключаем **Doctrine ORM** — и книги переезжают в настоящую базу данных.

## Что такое ORM

ORM (Object-Relational Mapping) связывает PHP-классы с таблицами базы данных: экземпляр класса
`Book` соответствует строке в таблице `book`, а свойство `$title` — столбцу `title`. Вместо того
чтобы писать SQL руками, ты работаешь с обычными PHP-объектами, а Doctrine сама генерирует нужные
запросы.

## Entity — класс, помеченный как сущность

```php
namespace App\Entity;

use Doctrine\ORM\Mapping as ORM;

#[ORM\Entity]
class Book
{
    #[ORM\Id]
    #[ORM\GeneratedValue]
    #[ORM\Column]
    private ?int $id = null;

    #[ORM\Column(length: 255)]
    private string $title = '';

    #[ORM\Column]
    private int $year = 0;

    // геттеры и сеттеры — обычный PHP, ничего специального
}
```

Разберём атрибуты:

- **`#[ORM\Entity]`** над классом — «эта сущность управляется Doctrine, под неё нужна таблица».
- **`#[ORM\Id]`** — это первичный ключ.
- **`#[ORM\GeneratedValue]`** — значение генерируется автоматически (обычно автоинкрементом в БД),
  вручную его задавать не нужно.
- **`#[ORM\Column]`** — обычное поле таблицы. Можно уточнить тип и длину:
  `#[ORM\Column(length: 255)]` для строки, `#[ORM\Column(type: 'text')]` для длинного текста.

## Создание схемы: schema:create vs миграции

```bash
php bin/console doctrine:schema:create
```

Эта команда смотрит на все Entity-классы и создаёт под них таблицы с нуля. Быстро и удобно для
разработки — именно её использует песочница этого курса. Но в реальном проекте, где в базе уже
есть данные, `schema:create` не годится — там нужны **миграции**
(`doctrine/doctrine-migrations-bundle`): версионируемые файлы с точными инструкциями «как изменить
схему, не потеряв существующие данные». Это тема для отдельного глубокого разбора, но на
собеседовании важно чётко понимать разницу и не путать одно с другим.

## Где хранятся данные в песочнице этого блока

В упражнениях этого блока используется настоящая **SQLite**-база — файл на диске внутри
одноразового контейнера. В отличие от PHP-массива в контроллере (Блок 1), эти данные переживают
границу одного HTTP-запроса: создал книгу одним запросом — она реально останется в базе для
следующего. Этим и займёмся в практике.
