Entity описывает *форму* данных. EntityManager — это то, что реально читает и пишет их в базу.

## persist() и flush() — два разных шага

```php
use Doctrine\ORM\EntityManagerInterface;

public function create(EntityManagerInterface $em): Response
{
    $book = new Book();
    $book->setTitle('Dune');
    $book->setYear(1965);

    $em->persist($book); // "запомни, что этот объект нужно будет сохранить"
    $em->flush();        // только теперь реально выполняется INSERT
}
```

Это разделение не случайно: между `persist()` и `flush()` можно накопить сколько угодно
изменений (создать несколько объектов, изменить существующие, удалить какие-то) — и отправить
всё одним проходом при вызове `flush()`. До `flush()` у объекта ещё даже нет `id` — он
появляется только после реального `INSERT`.

## Чтение: find() и findOneBy()

```php
$book = $em->getRepository(Book::class)->find(5);                    // по первичному ключу
$book = $em->getRepository(Book::class)->findOneBy(['title' => 'Dune']); // по любому полю
$books = $em->getRepository(Book::class)->findAll();                  // все записи
$books = $em->getRepository(Book::class)->findBy(['year' => 1965]);   // все, подходящие под условие
```

Все эти методы уже есть «из коробки» — Doctrine генерирует базовый репозиторий автоматически,
даже если ты не написал ни строчки кода для него.

## identity map: одна и та же строка — один и тот же объект

Внутри одного запроса Doctrine кэширует уже загруженные объекты — два вызова `find(5)` подряд
вернут буквально один и тот же PHP-объект в памяти, без повторного похода в базу. Это удобно, но
может маскировать баги при тестировании: если нужно гарантированно перечитать данные именно из
базы (а не из этого кэша) — вызывают `$em->clear()`, которая сбрасывает identity map.

```php
$em->persist($book);
$em->flush();
$id = $book->getId();

$em->clear(); // "забудь всё, что помнишь"
$fresh = $em->getRepository(Book::class)->find($id); // честно читает из базы заново
```

Именно так устроены проверки в упражнениях этого блока — они специально вызывают `clear()`,
чтобы удостовериться, что данные реально попали в базу, а не просто остались в памяти PHP.
