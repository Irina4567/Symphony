Есть два способа создать новый проект Symfony: через **Symfony CLI** (рекомендуемый) или
напрямую через **Composer**.

## Symfony CLI

Symfony CLI — это отдельный бинарник (не PHP-пакет), который управляет проектами Symfony:
создаёт их, поднимает локальный HTTPS-сервер, проверяет требования окружения и т.д.

```bash
# полноценное веб-приложение (Twig, security, форма, mailer и т.д.)
symfony new my_project --webapp

# минимальный skeleton — то, с чего обычно начинают API
symfony new my_project

cd my_project
symfony serve
```

Без флага `--webapp` CLI ставит только самый необходимый набор пакетов — это удобная отправная
точка для API, где не нужны шаблоны и формы.

## Через Composer

Если не хочется ставить отдельный CLI, то же самое можно сделать через `composer create-project`:

```bash
composer create-project symfony/skeleton my_project
# или полноценное приложение:
composer create-project symfony/webapp-skeleton my_project
```

## Structure проекта

После создания проект выглядит примерно так:

| Каталог          | За что отвечает                                                          |
| ---------------- | -------------------------------------------------------------------------- |
| `bin/`            | `bin/console` — точка входа для CLI-команд                               |
| `config/`         | YAML/PHP-конфигурация: сервисы, роуты, пакеты (`config/packages/*.yaml`) |
| `public/`         | Единственный каталог, доступный из браузера. Точка входа — `public/index.php` |
| `src/`            | Весь ваш PHP-код: контроллеры, сущности, сервисы, репозитории            |
| `templates/`      | Twig-шаблоны                                                              |
| `translations/`   | Файлы локализации                                                         |
| `var/`            | Генерируемые данные: `var/cache`, `var/log` — можно смело удалять, всё пересоберётся |
| `vendor/`         | Зависимости, поставленные Composer — сюда никогда не коммитят и не редактируют вручную |

![Структура каталогов проекта: bin, config, public, src, templates, var, vendor](/images/intro/project-structure.png)

Обратите внимание: **только `public/`** должен быть document root веб-сервера. Если открыть
из браузера `src/` или `config/` напрямую — это будет дыра в безопасности; поэтому все остальные
каталоги физически лежат вне зоны, которую отдаёт веб-сервер.

![public/index.php — единственная дверь наружу, всё остальное недоступно из браузера](/images/intro/public-dir-only-door.png)

## Symfony Flex

Когда вы выполняете `composer require <пакет>`, Symfony Flex (плагин Composer, который стоит по
умолчанию) находит для этого пакета **рецепт** — готовый набор конфигурационных файлов — и
автоматически раскладывает его по проекту: создаёт `config/packages/<пакет>.yaml`, добавляет
переменные в `.env`, и т.д. Именно поэтому в Symfony установка нового бандла обычно не требует
ручной конфигурации.
