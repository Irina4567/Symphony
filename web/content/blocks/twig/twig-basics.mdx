До сих пор BookShelf отдавал только JSON. В этом блоке добавляем HTML-витрину — и для этого
Symfony использует **Twig**, шаблонизатор, который стоит по умолчанию в любом full-stack проекте.

## Зачем шаблонизатор, если есть PHP

Технически можно писать HTML прямо в контроллере через `echo` и `<?php ?>` — но довольно быстро
это превращается в нечитаемую кашу, где логика и разметка перемешаны, а забыть экранировать
пользовательский ввод (и получить XSS) — вопрос времени. Twig решает обе проблемы: у него
специально урезанный, простой синтаксис (в шаблоне физически невозможно написать
произвольный PHP-код), и вывод **экранируется автоматически**.

## Рендеринг из контроллера

```php
use Symfony\Bundle\FrameworkBundle\Controller\AbstractController;

class BookController extends AbstractController
{
    #[Route('/books', name: 'books_list')]
    public function list(): Response
    {
        return $this->render('books/list.html.twig', [
            'books' => $this->bookRepository->findAll(), // пример — реальный репозиторий будет в блоке про Doctrine
        ]);
    }
}
```

`render()` — это хелпер `AbstractController`, который под капотом берёт сервис `twig`, находит
шаблон по пути (шаблоны лежат в `templates/`, файлы называются `*.html.twig`) и возвращает уже
готовый `Response` с отрендеренным HTML. В упражнениях этого блока контроллеры уже написаны за
тебя — ты пишешь только сами `.twig`-файлы, чтобы сосредоточиться на синтаксисе шаблонов.

## Два вида тегов

- **`{{ выражение }}`** — вывод значения. Автоматически экранирует HTML-спецсимволы.
- **`{% инструкция %}`** — управляющие конструкции: циклы, условия, наследование. Сами по себе
  ничего не выводят.
- **`{# комментарий #}`** — не попадает в итоговый HTML вообще, даже в закомментированном виде.

## Фильтры

Фильтр — это функция, применяемая к значению через `|`:

```twig
{{ title|upper }}                 {# ЗАГЛАВНЫМИ #}
{{ description|length }}          {# длина строки/массива #}
{{ price|number_format(2) }}      {# 99.50 #}
{{ published_at|date('d.m.Y') }}  {# форматирование даты #}
```

Фильтры можно комбинировать цепочкой: `{{ name|trim|upper }}`.

## Автоэкранирование — по умолчанию включено

```twig
{{ user_comment }}
```

Если `user_comment` содержит `<script>alert(1)</script>`, Twig выведет его как текст
(`&lt;script&gt;...`), а не выполнит как HTML. Отключить это можно фильтром `|raw` — но делать
это стоит только когда ты точно доверяешь источнику данных (например, содержимое пришло не от
пользователя, а сгенерировано внутри самого приложения).
