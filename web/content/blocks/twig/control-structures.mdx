Список книг без цикла не показать — разбираемся с управляющими конструкциями Twig.

## Цикл for

```twig
<ul>
{% for book in books %}
    <li>{{ book.title }} — {{ book.author }}</li>
{% endfor %}
</ul>
```

Обрати внимание: `book.title` — точечная нотация работает и для свойств объекта, и для ключей
массива одинаково. Twig сам разберётся, что перед ним — `$book['title']` или `$book->getTitle()`
(и даже вызовет геттер, если `title` — не публичное свойство).

## Ветка else — пустая коллекция

```twig
{% for book in books %}
    <li>{{ book.title }}</li>
{% else %}
    <li>Книг пока нет</li>
{% endfor %}
```

Это ровно та ситуация, которую полезно обработать явно — пустой каталог не должен выглядеть как
белый экран без объяснений.

## Переменная loop

Внутри `{% for %}` доступна специальная переменная `loop` с полезными свойствами:

```twig
{% for book in books %}
    {{ loop.index }}. {{ book.title }}   {# 1, 2, 3... #}
    {% if loop.first %}(первая в списке){% endif %}
    {% if loop.last %}(последняя в списке){% endif %}
{% endfor %}
```

`loop.index` считает с 1, `loop.index0` — с 0, `loop.length` — общее число элементов.

## Условия if / elseif / else

```twig
{% if book.year > 2020 %}
    <span class="badge">Новинка</span>
{% elseif book.year > 2000 %}
    <span class="badge">Недавняя</span>
{% else %}
    <span class="badge">Классика</span>
{% endif %}
```

Синтаксис условий похож на PHP alternative syntax (`if: ... elseif: ... else: ... endif;`), но
с `{% %}` вместо `<?php ?>`.
