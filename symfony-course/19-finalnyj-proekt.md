## 19.1. BookNest: полная спецификация проекта

Соберём воедино всё, что мы строили по модулям — это чеклист для самостоятельной финальной сборки проекта "с нуля до готового приложения", используя все 19 модулей курса.

### Функциональные требования

**Публичная часть (гости):**
- Каталог книг с пагинацией, поиском и фильтрами (модули 1, 5, 6).
- Страница книги с описанием, автором, категориями.
- Регистрация и логин покупателей (модуль 9).

**Личный кабинет покупателя (`ROLE_CUSTOMER`):**
- Корзина (модуль 4 — `CartManager`).
- Оформление заказа через форму (модуль 7) с валидацией (модуль 8).
- История заказов, отмена своего заказа (voter, модуль 10).
- Email-подтверждение заказа — асинхронно (модуль 14).

**Админ-панель (`ROLE_ADMIN`):**
- CRUD книг/авторов/категорий через формы (модуль 7).
- Импорт книг из CSV консольной командой (модуль 15).
- Просмотр всех заказов, изменение статуса.

**API (для мобильного приложения/интеграций):**
- REST API каталога через API Platform с фильтрами (модуль 12).
- JWT/токен-аутентификация для API (модуль 9, расширенно).
- Кастомная операция "отменить заказ" через API (модуль 12).

**Технический слой:**
- Кэширование каталога с тегами (модуль 15).
- Живой поиск и корзина без перезагрузки страницы (модуль 16).
- Полное покрытие тестами: unit для сервисов/voters, functional для контроллеров (модуль 13).
- CI-пайплайн с тестами и PHPStan (модуль 17, 18).
- Production-ready Docker-сборка и деплой-скрипт (модуль 17).

### Структура сущностей (итоговая ER-модель)

```
Author 1───* Book *───* Category
                │
                │ *
                ▼
             OrderItem *───1 Order *───1 User
```

### Порядок сборки "с чистого листа" (рекомендуемая последовательность)

1. Установка окружения, создание проекта (модуль 0).
2. Сущности `Book`, `Author`, `Category` + миграции (модуль 5).
3. Публичный каталог: роуты + Twig (модули 1, 2).
4. Связи, устранение N+1, продвинутый Doctrine (модуль 6).
5. `User`, регистрация, логин (модуль 9).
6. Роли, voters для заказов (модуль 10).
7. `Order`/`OrderItem`, форма оформления заказа, валидация (модули 7, 8).
8. Сервисный слой: `CartManager`, `PriceCalculator`, `CheckoutService` (модули 4, 18).
9. События и асинхронная отправка email (модуль 14).
10. Админ-панель для управления каталогом (модули 7, 10).
11. REST API через API Platform (модуль 12).
12. Тесты — unit и functional (модуль 13).
13. Кэш и консольные команды (модуль 15).
14. Живой поиск и корзина через Symfony UX (модуль 16).
15. Docker, CI/CD, деплой-чеклист (модуль 17).
16. Финальный рефакторинг по практикам архитектуры (модуль 18).

## 19.2. Итоговый чеклист компетенций "Junior → Middle → Senior/PRO"

### Уровень Junior Symfony-разработчика

- [ ] Понимает цикл Request → Response, роутинг, контроллеры
- [ ] Пишет Twig-шаблоны с наследованием
- [ ] Создаёт простые сущности Doctrine и миграции
- [ ] Пользуется формами и базовой валидацией
- [ ] Настраивает простую аутентификацию по логину/паролю
- [ ] Понимает, что такое сервис и DI на базовом уровне (autowiring)

### Уровень Middle

- [ ] Уверенно работает с DI: теги, фабрики, `#[Autowire]`, `#[Target]`
- [ ] Оптимизирует Doctrine-запросы, решает N+1, работает с транзакциями и батчингом
- [ ] Пишет Voters для сложной авторизации
- [ ] Строит REST API вручную и через API Platform
- [ ] Пишет unit и functional тесты, использует test doubles осмысленно
- [ ] Использует события и Messenger для развязки логики и асинхронности
- [ ] Настраивает кэширование с инвалидацией по тегам
- [ ] Пишет и поддерживает консольные команды

### Уровень Senior/PRO

- [ ] Проектирует архитектуру приложения (слои, границы ответственности) под конкретные требования проекта, а не "по шаблону"
- [ ] Осознанно применяет DDD-паттерны там, где они оправданы, и не применяет там, где не оправданы
- [ ] Настраивает production-инфраструктуру: Docker, CI/CD, мониторинг, деплой-стратегии
- [ ] Диагностирует и устраняет проблемы производительности через профилирование (не гадает)
- [ ] Пишет и поддерживает переиспользуемые Symfony-бандлы для внутренних нужд команды
- [ ] Проводит код-ревью с фокусом на архитектурные решения, а не только синтаксис
- [ ] Осознанно выбирает между "сделать руками" и "взять готовое решение" (например, ручной REST API vs API Platform) исходя из требований проекта, а не моды
- [ ] Умеет объяснить новичку любую тему этого курса простыми словами (лучший тест на реальное понимание)

## 19.3. Что дальше — темы для углубления после курса

Курс покрывает core Symfony максимально полно, но экосистема больше. Дальнейшие направления роста:

- **Symfony Workflow** — формализованные конечные автоматы для сложных жизненных циклов сущностей (упоминался в модуле 18).
- **EasyAdmin** — быстрая генерация полноценных админ-панелей поверх Doctrine-сущностей.
- **Symfony Panther** — end-to-end тестирование через реальный headless-браузер.
- **Multi-tenancy паттерны** — если понадобится SaaS с изоляцией данных клиентов.
- **Event Sourcing / полноценный DDD** — для действительно сложных доменов (банкинг, логистика).
- **Symfony на Swoole/RoadRunner/FrankenPHP (worker mode)** — постоянно работающие PHP-процессы для повышения производительности, с важными нюансами (statefulness сервисов, см. модуль 4).
- **Микросервисная архитектура на Symfony** — использование только нужных компонентов, коммуникация через Messenger/очереди между сервисами.
- **Contributing в сам Symfony** — чтение исходного кода фреймворка (он на удивление читаем) — лучший способ по-настоящему понять "магию" изнутри.

### Официальные ресурсы для дальнейшего изучения

- Официальная документация: symfony.com/doc/current (всегда проверяйте актуальность на конкретную версию).
- SymfonyCasts — обучающие скринкасты от команды, стоящей за фреймворком.
- Блог и changelog Symfony — для отслеживания нововведений между релизами.
- Исходный код `symfony/symfony` на GitHub — для по-настоящему глубокого понимания.

## 19.4. Финальное практическое задание

Соберите BookNest полностью самостоятельно по спецификации из раздела 19.1, придерживаясь порядка сборки. По завершении пройдите по чеклисту:

1. `php bin/console debug:router` — все маршруты именованы и логичны.
2. `php bin/phpunit` — все тесты зелёные.
3. `vendor/bin/phpstan analyse` — 0 ошибок на выбранном уровне.
4. `/api` — документация API Platform открывается и все операции работают через "Try it out".
5. Полный сценарий вручную: регистрация → добавление в корзину → оформление заказа → получение email (проверьте через Mailtrap/`MAILER_DSN=null://null` + профайлер) → просмотр в истории заказов → отмена заказа.
6. Полный сценарий администратора: логин → импорт книг через CSV → редактирование книги → просмотр всех заказов.
7. `docker compose -f compose.prod.yaml up` — приложение поднимается и отвечает на `/health`.

Если все пункты выполнены — вы прошли путь от 0 до PRO по Symfony на практике, а не только в теории.

## 19.5. Заключение

Symfony — фреймворк, вознаграждающий за глубокое понимание: каждый следующий модуль этого курса опирался на предыдущие не случайно — DI (модуль 4) лежит в основе всего, включая формы, security и Messenger; события (модуль 14) — та же идея, что лежит в основе всего HttpKernel (модуль 0). Как только этот фундамент уложился в голове, любая новая часть экосистемы (новый бандл, новый компонент) осваивается по аналогии в разы быстрее.

Удачи в разработке на Symfony!
